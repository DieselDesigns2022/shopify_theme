# Diesel Designs Custom Shopify Theme Plan

## 1. Theme Purpose

The theme will be a premium, general-purpose Shopify theme:

* Created and sold exclusively by Diesel Designs
* Designed mainly for businesses selling physical products
* Also compatible with digital products, services, and mixed-product stores
* Easy for store owners to customize without coding
* Extremely simple for customers to browse and purchase from
* Built from scratch without copying another theme’s code or exact design
* Designed to reduce the number of paid Shopify apps a store owner needs
* Flexible enough to work with many different shop styles and product types

The theme should work well for businesses selling items such as:

* Shirts and apparel
* Tumblers and drinkware
* Handmade products
* Jewelry and accessories
* Boutique products
* Home décor
* Beauty products
* Candles and wax melts
* Personalized products
* Printed products
* Craft supplies
* Digital downloads
* Custom services

The theme will not be designed around one specific brand style. Store owners will be able to change the colors, fonts, images, spacing, buttons, banners, and other visual settings.

# 2. Required Diesel Designs Credit

Every installation of the theme will include the following footer credit:

**Website by Diesel Designs**

“Diesel Designs” will link to:

**https://www.dieseldesigns.co**

## Credit Placement

The credit will appear near the bottom of the footer, close to the copyright information.

Example:

© 2026 Store Name
Website by Diesel Designs

## Credit Appearance

The store owner will be able to control:

* Text color
* Footer background
* Font size within an approved range
* Text alignment

The actual wording and Diesel Designs link will remain part of the theme.

## Important Limitation

Because Shopify store owners receive access to their installed theme code, it is not possible to make the credit completely impossible to remove. A store owner with code access could manually delete or change it.

We can still make the credit:

* Included automatically
* Present in every footer layout
* Separate from normal editable footer text
* Difficult to remove accidentally through the Theme Editor
* Clearly required under the theme’s license terms

The theme license and customer documentation should state that the “Website by Diesel Designs” credit must remain unless the customer purchases a separate credit-removal license.

This creates a possible future upsell:

* Standard theme license includes the credit
* Extended theme license allows credit removal

# 4. Theme Architecture

The theme will use Shopify’s current theme system, including:

* Liquid
* JSON templates
* Sections
* Section blocks
* Reusable theme blocks where appropriate
* Snippets
* CSS
* Lightweight JavaScript
* Shopify metafields
* Shopify metaobjects
* Dynamic sources
* App blocks
* Shopify AJAX Cart API
* Shopify Predictive Search

JSON templates allow sections to be added, removed, and rearranged through the Theme Editor, while blocks allow smaller pieces of content to be reordered inside sections.

The theme will be built in a GitHub repository so all changes, versions, and releases can be tracked. Shopify provides direct GitHub integration for theme development.

# 16. Product-Option Limitations

The theme must clearly separate two types of options.

## Options That Can Be Handled by the Theme

The built-in fields can collect information that does not independently control Shopify inventory or pricing.

Examples:

* Name personalization
* Gift message
* Design notes
* File upload
* Optional checkbox
* Non-priced choices
* Requested completion date

## Options That Must Use Shopify Variants

A Shopify variant should be used when the choice changes:

* Price
* Inventory
* SKU
* Weight
* Product availability
* Fulfillment requirements

## Options That May Still Need an App or Separate Product

Complex features may require an app, separate add-on product, Shopify Function, or custom development, including:

* Adding different prices to many custom checkboxes
* Complex conditional pricing
* Live product previews
* Advanced product builders
* Large bundles with inventory tracking
* Measurement-based pricing
* Hundreds of unrelated paid add-ons

The theme should never claim to replace every product-options app. It should replace an app for common personalization and product-selection needs.

# 17. Conditional Product Fields

Where reasonably possible, custom fields may appear based on a customer’s selection.

Examples:

* Selecting “Yes, personalize it” reveals a name field
* Selecting “Other” reveals a text box
* Selecting “Upload my logo” reveals a file field
* Selecting “This is a gift” reveals a gift-message field

Conditional fields must remain simple and dependable.

They should not be used to fake inventory, change official variant pricing, or create complicated product logic that Shopify cannot safely process.

* Avoid unnecessary animation libraries
* Avoid loading code for disabled features
* Minimize large CSS and JavaScript files
* Prevent layout shifting
* Load essential content first
* Keep product forms usable when JavaScript fails where reasonably possible

Shopify’s current Theme Store benchmark requires a minimum average Lighthouse performance score of 60 on desktop and mobile across the home, product, and collection pages.

Our preferred internal target should be higher than the minimum whenever the demo content and Shopify environment allow it.
